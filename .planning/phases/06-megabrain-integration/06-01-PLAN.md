```python
"""MIS/MEGABRAIN bridge — único ponto de integração entre os dois sistemas.

Funções públicas:
    get_briefing_data()        → dict com dados para /mis-briefing
    export_to_megabrain(dest)  → dict com resultado da exportação
"""
import asyncio, hashlib, json, os, pathlib, structlog
from datetime import datetime, timedelta

log = structlog.get_logger(__name__)

# Chamada idempotente no import — garante DB atualizado
def _init_db():
    db_path = os.environ.get("MIS_DB_PATH", "data/mis.db")
    try:
        from mis.db import run_migrations
        run_migrations(db_path)
    except Exception:
        pass

_init_db()
```

**get_briefing_data():**
- Ler MIS_DB_PATH do environ (default "data/mis.db")
- Chamar `list_dossiers_by_rank(db_path, order_by='score', order_dir='desc', per_page=10)` para top-10
- Para cada nicho em `load_config().settings.niches`: chamar `get_latest_report(db_path, niche.id)`, extrair top-5 pains da lista `report['pains'][:5]`, incluir niche.language
- Chamar `get_unseen_count(db_path)` para alertas
- Calcular health score (0-100): scrapers OK=40pts (asyncio.run(run_canary_check()) ? 40 : 0), último ciclo <2h=30pts (verificar max(created_at) de dossiers), dossiês hoje>0=20pts, alertas sem falha crítica=10pts
- Retornar dict completo com campo `last_cycle` (ISO timestamp do produto mais recente ou None) e `data_stale` (bool, True se >2h)
- Tratar TODAS as exceções: retornar `{'status': 'error', 'message': str(e), 'setup_hint': 'Execute python -m mis para inicializar.'}`

**export_to_megabrain(dest=None):**
- Resolver dest: parâmetro > env MEGABRAIN_PATH/knowledge/mis > erro
- Criar dest se não existir (`pathlib.Path(dest).mkdir(parents=True, exist_ok=True)`)
- Query dossiês completos: `SELECT d.*, p.name, p.external_id, pl.slug as platform_slug, n.slug as niche_slug FROM dossiers d JOIN products p ON p.id=d.product_id JOIN platforms pl ON pl.id=p.platform_id JOIN niches n ON n.id=p.niche_id WHERE d.status='complete'`
- Para cada dossiê: gerar nome `dossier_{platform_slug}_{p.external_id}.md`, calcular MD5 do conteúdo Markdown, comparar com arquivo existente (se mesmo hash, skip)
- Formato Markdown: frontmatter YAML (`source: mis`, `type: dossier`, `platform`, `niche`, `score`, `date`, `product_id`) + seções `## Por que Vende`, `## Copy`, `## Anúncios`, `## Reviews`, `## Template`
- Pain reports dos últimos 7 dias: query `SELECT pr.*, n.slug as niche_slug FROM pain_reports pr JOIN niches n ON n.id=pr.niche_id WHERE pr.generated_at >= ?`; nome `pain_{niche_slug}_{YYYYMMDD}.md`
- Gerar/atualizar `{dest}/README.md` com data export e contagens
- Retornar `{'status': 'ok', 'exported': N, 'skipped': M, 'dest': str(dest)}`
- structlog em cada arquivo: `log.info('export_file', file=path, type='dossier'|'pain_report')`
