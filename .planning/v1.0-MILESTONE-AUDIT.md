## Integration Defects

### DEFECT-1 ⚠ Critical — niche_id=0 in all products

**Affected:** SCAN-01, SCAN-02, SCAN-03, DASH-01
**Location:** `mis/scanner.py` lines 200–208

`run_all_scanners()` instantiates scanners without passing `niche_id`. `config.yaml` niche entries only have `slug`/`name`/`platforms` — no `id` field. All products are saved with `niche_id=0`. The JOIN in `list_dossiers_by_rank()` (`JOIN niches n ON n.id = p.niche_id`) returns zero rows. Dashboard ranking page is permanently empty in production.

**Fix:** Before calling `scanner_cls(...)`, resolve `niche_id` via `SELECT id FROM niches WHERE slug=?`.

---

### DEFECT-2 — register_scanner_jobs() orphaned (Low)

**Affected:** SCAN-04 (partially)
**Location:** `mis/web/app.py` lifespan

The lifespan calls `register_scan_and_spy_job()` (combined scan+spy) but never `register_scanner_jobs()` (3 individual platform jobs). Individual platform jobs are dead code at runtime. Combined job provides equivalent coverage so SCAN-04 works in practice.

**Fix:** Document that `register_scan_and_spy_job` supersedes individual jobs, or remove `register_scanner_jobs()`.

---

### DEFECT-3 ⚠ Critical — Radar jobs fail with asyncio.run() inside AsyncIOScheduler

**Affected:** RADAR-01, RADAR-02, RADAR-03, RADAR-05, RADAR-06
**Location:** `mis/radar/__init__.py` lines 149–162

All 5 radar job wrappers call `asyncio.run()` (sync pattern) inside `AsyncIOScheduler` which runs on uvicorn's event loop. This raises `RuntimeError: This event loop is already running` on every trigger. The Pain Radar will never produce data in production.

**Fix:** Convert all radar sync wrappers to `async def`:
```python
# Before (broken):
def _trends_job():
    asyncio.run(collect_niche_trends(config))

# After (correct):
async def _trends_job():
    await collect_niche_trends(config)
```
