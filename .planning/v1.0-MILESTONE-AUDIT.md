## Cross-Phase Integration Issues

### ❌ BUG-01: export_to_megabrain() — Status Filter Mismatch

**File:** `mis/mis_agent.py:308`
**Impact:** `export_to_megabrain()` queries `WHERE d.status = 'complete'` but `spy_orchestrator.py` writes `status = 'done'`. Result: function always exports **0 dossiers** silently.
**Affected:** INT-01, INT-02

### ❌ BUG-02: created_at Column Does Not Exist in Dossiers Table

**File:** `mis/mis_agent.py:149, 229`
**Impact:** `get_briefing_data()` queries `MAX(created_at)` and `WHERE created_at >= ?` but the column is named `generated_at` (per `_001_initial.py`). Errors swallowed silently. Health score always loses 50 pts (`cycle_fresh` + `dossiers_today`).
**Affected:** FOUND-04, INT-01, INT-02

### ❌ BUG-03: _scan_and_spy_job() Never Saves Products

**File:** `mis/scheduler.py:89-110`
**Impact:** `_scan_and_spy_job` calls `run_all_scanners()` which returns in-memory `Product` objects (no DB IDs). All products are skipped at the `no_db_id` check. Spy pipeline never triggers. Individual `register_scanner_jobs()` path saves correctly but doesn't chain into spy.
**Affected:** SCAN-04 (automated path), all SPY/DOS requirements via automated pipeline, DASH-02, DASH-04
