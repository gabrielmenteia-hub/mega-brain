    - Se ?niche= e ?subniche= nos query params, pré-popular select (passados como context)

    POST /search/run (Form: subniche_id: int):
    - Verificar se há sessão running para subniche_id: `sessions = list_recent_sessions(db_path); running = next((s for s in sessions if s["subniche_id"] == subniche_id and s["status"] == "running"), None)`
    - Se running: redirecionar para /search/{running["id"]}/status (sem criar nova sessão)
    - Caso contrário: create_session(db_path, subniche_id) → session_id
    - asyncio.create_task(run_manual_search(session_id, subniche_id, db_path))
    - register_task(session_id, task)
    - return RedirectResponse(f"/search/{session_id}/status", status_code=302)

    GET /search/{session_id}/status:
    - get_session(db_path, session_id) → se status in ("done", "timeout"): redirect 302 para /search/{id}/results
    - Caso contrário: renderizar search_status.html com session + subnicho name (JOIN via get_session)

    GET /search/{session_id}/status/poll:
    - get_session(db_path, session_id)
    - Se done/timeout: response com HX-Redirect header para /search/{id}/results
    - Caso contrário: renderizar search_status_poll.html com session

    GET /search/{session_id}/results:
    - get_session(db_path, session_id)
    - platform_filter = request.query_params.get("platform")
    - products = list_session_products(db_path, session_id, platform_filter=platform_filter, per_page=20, page=1)
    - list_niches para buscar niche do subnicho (para breadcrumb)
    - Renderizar search_results.html ou search_results_table.html (se is_htmx())

    GET /search/{session_id}/results/table:
    - Mesmo que acima mas SEMPRE retorna partial search_results_table.html

    DELETE /search/{session_id}:
    - cancel_task(session_id)  # cancela task asyncio se running
    - delete_session(db_path, session_id)
    - response = RedirectResponse("/pesquisar", status_code=302)
    - response.headers["HX-Trigger"] = '{"showToast": "Pesquisa removida"}'
    - return response
