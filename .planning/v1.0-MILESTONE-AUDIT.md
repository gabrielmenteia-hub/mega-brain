## Cross-Phase Integration Findings

### Wiring Gaps (Critical)

#### INT-GAP-01: Scheduler not started from production entry point
**Affects:** SCAN-04, RADAR-01 through RADAR-06 (automated cycles)

- `register_scanner_jobs()`, `register_scan_and_spy_job()`, `register_radar_jobs()`, and `start_scheduler()` are defined and correct but **never called from any production entry point**
- `mis/web/app.py create_app()` has no FastAPI lifespan hook
- `mis/__main__.py _handle_dashboard()` does not register jobs before `uvicorn.run()`
- Result: automated daily scans, spy pipeline, and hourly radar will NOT run when server starts
- **Manual CLI invocations work fine** — only the scheduled automation is broken

**Fix:** Add FastAPI lifespan context manager to `create_app()`:
```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    config = load_config()
    register_scanner_jobs(config)
    register_scan_and_spy_job(config)
    register_radar_jobs(config)
    register_canary_job()
    get_scheduler().start()
    yield
    get_scheduler().shutdown()
```

#### INT-GAP-02: proxy_list not forwarded through PlatformScanner
**Affects:** FOUND-02 (proxy rotation)

- `scanner.py:206` calls `scanner_cls(proxy_url=proxy_url, proxy_list=proxy_list)`
- `PlatformScanner.__init__` at `scanner.py:63` only accepts `proxy_url` — **TypeError raised** at runtime if `proxy_list` is non-empty
- `PlatformScanner._scraper = BaseScraper(proxy_url=proxy_url)` at `scanner.py:68` — `proxy_list` not forwarded even if accepted
- Currently safe only because `config.yaml` has `proxy_list: []` (default)

**Fix:**
```python
# scanner.py line ~63
def __init__(self, proxy_url: Optional[str] = None, proxy_list: Optional[list[str]] = None):
    self._scraper = BaseScraper(proxy_url=proxy_url, proxy_list=proxy_list)
```
Plus same update to all 3 concrete scanner `__init__` + `super()` calls.
